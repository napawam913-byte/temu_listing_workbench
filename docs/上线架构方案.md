# Temu 选品上架工作台上线架构方案

## 1. 目标

本项目当前目标不是一次性做成复杂分布式系统，而是按阶段上线：

1. 先把核心数据迁到云 PostgreSQL。
2. 本地前后端连接云数据库完成业务验证。
3. 云服务器部署前端、后端和 Nginx。
4. 后续再接 Redis + Worker，承接导出、生图、批量 AI 分析等长任务。

核心原则：

- PostgreSQL 作为唯一主数据库。
- 已迁移模块不再回退 SQLite，避免测试混淆。
- Redis 暂时不是必须，等长任务队列稳定后再接入。
- OSS 用于图片、导出文件、生图结果等对象存储。

## 2. 当前测试架构

```text
本地浏览器
   |
本地前端 Vite: 127.0.0.1:5173
   |
本地 FastAPI 后端: 127.0.0.1:8000
   |
阿里云 PostgreSQL 云数据库
```

当前阶段用于验证：

- 云数据库迁移是否完整。
- 数据台、商品池是否真实读取云端数据。
- 登录、用户、团队是否真实写入云端。
- 本地开发体验是否仍然稳定。

目前已经硬切云 PostgreSQL 的模块：

```text
数据台 / 商品池
登录会话 / 用户 / 团队
```

这些模块现在不再通过开关回退 SQLite。如果云 PostgreSQL 连接或数据有问题，接口会直接报错，方便定位。

## 3. 正式上线第一版架构

```text
用户浏览器
   |
域名 + HTTPS
   |
Nginx
   |
React 前端静态文件
   |
FastAPI 后端服务
   |
阿里云 PostgreSQL
   |
阿里云 OSS
```

第一版上线重点：

- 前端 build 后由 Nginx 托管。
- Nginx 反向代理 `/api` 到 FastAPI。
- FastAPI 连接阿里云 PostgreSQL。
- OSS 存储图片、导出文件、生图结果。
- 暂时不强依赖 Redis。

第一版需要保证稳定的业务：

```text
用户登录
团队与用户管理
数据台筛选
商品池管理
1688 货源采集
导出基础流程
AI 配置读取
图片上传与访问
```

## 4. 后续增强版架构

```text
用户浏览器
   |
Nginx / HTTPS
   |
React 前端
   |
FastAPI API 服务
   |
PostgreSQL 主数据库
   |
Redis 队列 / 锁 / 任务状态
   |
后台 Worker
   |
OSS 图片与导出文件
   |
OpenAI / 初凡AI / 1688 / 店小秘
```

增强版加入 Redis 和 Worker，主要解决长任务与并发问题。

Redis 适合负责：

```text
生图任务队列
导出清单后台任务
任务失败重试
并发锁
多后端实例共享任务状态
实时进度同步
```

Worker 适合负责：

```text
批量导出清单
批量标题生成
批量产品属性填写
批量生图
批量 1688 素材处理
失败任务重试
```

## 5. 组件职责

### PostgreSQL

PostgreSQL 是业务主数据库，负责保存结构化数据：

```text
商品
商品池
用户
团队
登录会话
店小秘类目
店小秘属性库
1688 货源
链接列表
AI 调用日志
导出任务
后台配置
API Key / 模型配置
```

### FastAPI

FastAPI 是业务后端，负责：

```text
登录鉴权
用户与团队权限
商品查询筛选
商品池管理
1688 采集与推荐
导出清单生成
AI 标题生成
AI 类目匹配
产品属性填写
生图任务调度
```

### React 前端

React 负责业务操作界面：

```text
数据台
商品池
链接列表
1688 采集面板
SKU 选择
清单导出
后台配置
任务队列展示
```

### Nginx

Nginx 负责上线入口：

```text
HTTPS
域名访问
前端静态文件托管
/api 反向代理到 FastAPI
大文件上传限制
请求超时配置
```

### OSS

OSS 负责对象存储：

```text
商品图片
1688 素材图片
AI 生图结果
导出 Excel 文件
临时下载文件
```

### Redis

Redis 不是当前 MVP 必需。后期用于：

```text
后台任务队列
任务状态同步
分布式锁
失败重试
并发控制
```

## 6. 推荐上线顺序

### 第一阶段：本地前后端连接云 PostgreSQL

目标：

- 本地前端、本地后端继续运行。
- 核心数据从云 PostgreSQL 读取和写入。
- 验证数据迁移完整性。

当前状态：

```text
数据台 / 商品池：已硬切云 PostgreSQL
登录会话 / 用户 / 团队：已硬切云 PostgreSQL
```

### 第二阶段：继续迁移剩余配置与日志模块

建议下一批迁移：

```text
app_settings
user_api_settings
user_api_credentials
user_usage_limits
api_usage_logs
export_product_attribute_jobs
dianxiaomi_export_tasks
visual_generation_tasks
creative_image_jobs
```

这一步完成后，后台配置、API Key、AI 模型配置、导出任务和生图任务会逐步从 SQLite 脱离。

### 第三阶段：云服务器部署后端

目标：

- 在云服务器部署 FastAPI。
- 使用生产环境 `.env`。
- 后端连接阿里云 PostgreSQL。
- 后端日志输出到文件或进程管理器。

建议进程管理方式：

```text
systemd
或
Supervisor
或
Docker Compose
```

### 第四阶段：部署前端与 Nginx

目标：

- 前端执行 build。
- Nginx 托管前端静态文件。
- Nginx 将 `/api` 转发到 FastAPI。
- 绑定域名和 HTTPS。

示意：

```text
https://your-domain.com
   |
Nginx
   |-- /        -> React 静态文件
   |-- /api     -> FastAPI:8000
```

### 第五阶段：OSS 正式化

目标：

- 商品图、生图图、导出文件统一上传 OSS。
- 数据库只保存 OSS URL。
- 本地 `storage` 只作为临时目录。

### 第六阶段：接入 Redis + Worker

目标：

- 导出清单不再阻塞弹窗。
- 生图任务进入后台队列。
- 任务支持进度查询、失败重试、并发限制。
- 多个后端实例可以共享任务状态。

## 7. 当前不需要马上租 Redis 的原因

现在只有这些条件时，PostgreSQL 足够：

```text
本地前后端测试
单个后端进程
数据量处于 MVP 阶段
长任务还没有完全队列化
```

Redis 应该在以下情况出现后再接：

```text
导出清单需要后台持续运行
生图任务数量明显增多
需要任务失败自动重试
准备多进程或多实例部署 FastAPI
需要任务中心实时显示状态
```

## 8. 验证清单

上线前需要逐项验证：

```text
用户可以登录
刷新页面后仍然保持登录
用户列表来自云 PostgreSQL
团队归属正确
数据台商品总数正确
商品池数量正确
类目一级数量正确
商品筛选正常
加入商品池后云库有记录
删除商品后云库状态变化
后台配置可读取
导出任务可创建
OSS 图片可访问
AI 调用日志可记录
```

## 9. 最终目标架构

最终形态：

```text
Nginx + React + FastAPI + PostgreSQL + OSS + Redis + Worker
```

阶段策略：

```text
先 PostgreSQL 单主库上线
再 Redis + Worker 扩展长任务能力
最后按流量和任务量扩容后端实例
```
