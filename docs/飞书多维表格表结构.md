# 飞书多维表格表结构

这份结构按前端真实 `LinkListRecord` JSON 设计，用于把工作台链接列表中的商品同步到飞书，再由飞书完成生图。

核心流程只保留三步：

```text
1. 根据原始产品图生成白底图
2. 根据白底图 + 前端 JSON 分析并生成当前图片提示词
3. 根据当前图片提示词生成结果图
```

不再设计复杂的主商品表、任务表、回传表。第一阶段只用一张表即可。

## 推荐表

表名：`生图任务表`

一行代表一张要生成的图片。

例如一个链接商品要生成 8 张主图，就插入 8 行；如果只想重做第 3 张图，就只插入 1 行。

同一个商品的多行任务通过 `链接记录ID` 关联。白底图可以第一行生成，后续行直接复用 `白底图URL`。

## 前端 JSON 来源

前端类型在：

```text
frontend/src/types/linkList.ts
```

核心结构：

```ts
type LinkListRecord = {
  schemaVersion?: 3;
  id: string;
  createdAt: string;
  productId: string;
  productTitle: string;
  productTitleEn?: string;
  mainImage?: LinkListImageAsset;
  productMaterialImages?: LinkListImageAsset[];
  imageSlots?: LinkListImageSlot[];
  styleProfile?: LinkListStyleProfile;
  productImageUrl?: string;
  productSourceUrl?: string;
  sourceLinks: LinkListSource[];
  skuEntries: LinkListSkuEntry[];
  componentSkuCount: number;
  creativeJobs?: LinkListCreativeJobSummary[];
};
```

飞书不需要把所有嵌套字段拆成很多列，只需要保留完整 JSON，再额外提取几个字段方便字段捷径读取。

## 字段结构

### 任务标识字段

| 字段名 | 飞书字段类型 | 必填 | 对应前端字段 | 说明 |
| --- | --- | --- | --- | --- |
| 任务ID | 文本 | 是 | 后端生成 | 唯一任务 ID，建议 `链接记录ID-图片类型-图片序号` |
| 链接记录ID | 文本 | 是 | `record.id` | 前端链接列表记录 ID |
| 商品ID | 文本 | 是 | `record.productId` | 商品 ID |
| 创建时间 | 日期时间 | 否 | `record.createdAt` | 工作台链接创建时间 |
| 状态 | 单选 | 是 | 后端/飞书维护 | `待处理`、`白底图完成`、`提示词完成`、`生图完成`、`失败` |
| 错误信息 | 多行文本 | 否 | 飞书/后端维护 | 失败时记录原因 |

### 商品输入字段

| 字段名 | 飞书字段类型 | 必填 | 对应前端字段 | 说明 |
| --- | --- | --- | --- | --- |
| 产品标题 | 文本 | 是 | `record.productTitle` | 中文标题 |
| 英文标题 | 文本 | 否 | `record.productTitleEn` | 店小秘导出用英文标题 |
| 原始产品图URL | URL / 文本 | 是 | `record.productImageUrl` 或 `record.mainImage.displayUrl` | 白底图输入图 |
| 原始产品图 | 附件 | 否 | 由 URL 下载后上传 | 如果飞书字段捷径不能直接读 URL，就用附件 |
| 前端完整JSON | 多行文本 | 是 | 整个 `LinkListRecord` | 保留完整数据，后续可回放 |
| 主图_JSON | 多行文本 | 否 | `record.mainImage` | 主图资产对象 |
| 素材图_JSON | 多行文本 | 否 | `record.productMaterialImages` | 1688 素材图/商品素材图 |
| 图片槽_JSON | 多行文本 | 否 | `record.imageSlots` | 前端预览页图片顺序 |
| 货源_JSON | 多行文本 | 否 | `record.sourceLinks` | 1688 货源链接和店铺信息 |
| SKU_JSON | 多行文本 | 否 | `record.skuEntries` | 全部销售 SKU 和组合 SKU |
| 组件SKU数量 | 数字 | 否 | `record.componentSkuCount` | SKU 组件总数 |

### 当前图片任务字段

这些字段决定当前这一行要生成哪张图。

| 字段名 | 飞书字段类型 | 必填 | 来源 | 说明 |
| --- | --- | --- | --- | --- |
| 图片序号 | 数字 | 是 | 后端生成 | 主图 1-8；SKU 图按 SKU 顺序 |
| 图片类型 | 单选 | 是 | 后端/人工选择 | 见下方图片类型 |
| 图片标签 | 文本 | 否 | 后端生成 | 例如 `主图`、`效果图`、`SKU-White` |
| 当前参考图URL | URL / 文本 | 否 | 主图或 SKU 图 | SKU 图优先用 SKU 自己的图 |
| SKU_ID | 文本 | 否 | `skuEntries[].id` | 只有 SKU 图需要 |
| SKU名称 | 文本 | 否 | `skuEntries[].name` | 只有 SKU 图需要 |
| SKU类型 | 单选 | 否 | `skuEntries[].kind` | `single` 或 `combo` |
| SKU组件_JSON | 多行文本 | 否 | `skuEntries[].componentSkus` | 组合 SKU 必填，告诉 AI 同框展示哪些商品 |
| SKU原图URL | URL / 文本 | 否 | `skuEntries[].imageUrl` 或 `imageAsset.displayUrl` | SKU 图参考图 |

### 三步生图字段

| 步骤 | 字段名 | 飞书字段类型 | 输入 | 输出 |
| --- | --- | --- | --- | --- |
| 第一步 | 白底图 | Seedream 4.0 / 附件 | `原始产品图` 或 `原始产品图URL` | 干净白底标准图 |
| 第一步 | 白底图URL | URL / 文本 | `白底图` | 后端或人工回填 |
| 第二步 | 生图提示词 | AI 文本 / 多行文本 | `白底图` + `前端完整JSON` + 当前任务字段 | 当前这一张图的英文提示词 |
| 第三步 | 生成结果图 | Seedream 4.0 / 附件 | `白底图` + `生图提示词` | 最终图片 |
| 第三步 | 生成结果图URL | URL / 文本 | `生成结果图` | 后端或人工回填 |
| 第三步 | OSS_URL | URL / 文本 | `生成结果图URL` | 工作台上传 OSS 后回填 |

## 图片类型

`图片类型` 建议做成单选字段。

| 值 | 用途 |
| --- | --- |
| 主图 | 商品首图 |
| 效果图 | 展示功能效果，不做夸大承诺 |
| 人物场景使用图 | 成人生活场景，避免儿童、名人、医疗场景 |
| 场景使用图 | 家居、桌面、户外等使用环境 |
| 细节图 | 材质、纹理、结构细节 |
| 尺寸结构图 | 尺寸和组件关系，文字必须简短安全 |
| 对比图 | 中性对比，不使用“最好、升级、保证”等词 |
| 套装变体图 | 展示套装、颜色、变体或组合内容 |
| SKU图 | 单个销售 SKU |
| 组合SKU图 | 多 SKU 或多货源组合出售 |

## 状态

`状态` 建议做成单选字段。

| 状态 | 含义 |
| --- | --- |
| 待处理 | 任务刚同步到飞书 |
| 白底图生成中 | 正在跑第一步 |
| 白底图完成 | 可以生成提示词 |
| 提示词生成中 | 正在跑第二步 |
| 提示词完成 | 可以生成结果图 |
| 生图中 | 正在跑第三步 |
| 生图完成 | 结果图已生成 |
| 已采用 | 人工确认采用 |
| 已废弃 | 人工确认不采用 |
| 已回传 | 已同步回工作台 |
| 失败 | 任务失败 |

## 前端字段映射

| 飞书字段 | 前端 JSON 路径 | 处理规则 |
| --- | --- | --- |
| 链接记录ID | `record.id` | 原样写入 |
| 商品ID | `record.productId` | 原样写入 |
| 产品标题 | `record.productTitle` | 原样写入 |
| 英文标题 | `record.productTitleEn` | 没有就留空 |
| 原始产品图URL | `record.productImageUrl` | 优先使用 |
| 原始产品图URL | `record.mainImage.editedCloudUrl` | 如果已有改图，优先级高于原图 |
| 原始产品图URL | `record.mainImage.displayCloudUrl` | 其次 |
| 原始产品图URL | `record.mainImage.displayUrl` | 其次 |
| 原始产品图URL | `record.mainImage.sourceUrl` | 兜底 |
| 主图_JSON | `record.mainImage` | `JSON.stringify` |
| 素材图_JSON | `record.productMaterialImages` | `JSON.stringify` |
| 图片槽_JSON | `record.imageSlots` | `JSON.stringify` |
| 货源_JSON | `record.sourceLinks` | `JSON.stringify` |
| SKU_JSON | `record.skuEntries` | `JSON.stringify` |
| 组件SKU数量 | `record.componentSkuCount` | 数字 |
| 前端完整JSON | `record` | `JSON.stringify(record)` |
| SKU_ID | `skuEntry.id` | 只有 SKU 图任务写入 |
| SKU名称 | `skuEntry.name` | 只有 SKU 图任务写入 |
| SKU类型 | `skuEntry.kind` | `single` 或 `combo` |
| SKU组件_JSON | `skuEntry.componentSkus` | `JSON.stringify` |
| SKU原图URL | `skuEntry.imageUrl` | 优先使用 |
| SKU原图URL | `skuEntry.imageAsset.editedCloudUrl` | 如果已有改图，优先级高于原图 |
| SKU原图URL | `skuEntry.imageAsset.displayCloudUrl` | 其次 |
| SKU原图URL | `skuEntry.imageAsset.displayUrl` | 其次 |
| SKU原图URL | `skuEntry.imageAsset.sourceUrl` | 兜底 |

## 前端同步一行的最小 JSON

飞书第一阶段只需要这些字段就能跑通：

```json
{
  "任务ID": "link-entry-xxx-main-1",
  "链接记录ID": "link-entry-xxx",
  "商品ID": "606559992167681",
  "产品标题": "1.85加仑大容量智能304不锈钢狗狗饮水机...",
  "英文标题": "1.85-Gallon Large Capacity Smart Stainless Steel Pet Water Fountain...",
  "原始产品图URL": "https://img.kwcdn.com/product/fancy/example.jpg",
  "前端完整JSON": "{...完整 LinkListRecord JSON...}",
  "SKU_JSON": "[...record.skuEntries...]",
  "货源_JSON": "[...record.sourceLinks...]",
  "图片序号": 1,
  "图片类型": "主图",
  "图片标签": "主图",
  "状态": "待处理"
}
```

如果是 SKU 图任务，再补：

```json
{
  "SKU_ID": "single-source-1-sku-1",
  "SKU名称": "White",
  "SKU类型": "single",
  "SKU原图URL": "https://example.com/sku-white.jpg",
  "SKU组件_JSON": "[...]",
  "图片类型": "SKU图"
}
```

如果是组合 SKU 图任务，`SKU组件_JSON` 必须包含组合内全部商品组件，提示词里必须要求同框展示所有购买会收到的商品。

## 第一步：白底图字段捷径

字段：`白底图`

类型：`Seedream 4.0`

图片输入：

```text
优先选：原始产品图
如果没有附件能力，再用：原始产品图URL
```

提示词：

```text
请先分析输入商品图片，再执行生成。

任务：将原始商品图片处理成干净的电商白底标准图。

要求：
1. 保持商品本身的造型、颜色、材质、结构、比例和可见细节不变。
2. 去除原图背景、中文文字、水印、店铺logo、平台标识、促销贴纸、价格、评分、二维码。
3. 使用纯白或极浅灰白背景。
4. 商品居中展示，构图干净，适合后续作为图生图参考图。
5. 不要新增不存在的配件。
6. 不要改变商品颜色。
7. 不要生成模特、人物、场景。
8. 不要出现品牌名、平台名、认证词、医疗功效词、夸大营销词。
9. 输出方形 1:1 电商产品白底图。

商品标题：{{产品标题}}
参考图片：{{原始产品图}}
```

## 第二步：生图提示词字段捷径

字段：`生图提示词`

类型：`Custom AI Autofill` / AI 文本

这个字段只生成当前这一行对应图片的提示词，不一次性生成所有图片提示词。

提示词：

```text
你是跨境电商图片提示词工程师。

请先分析，再输出当前这一张图片的英文生图提示词。

输入数据：
产品标题：{{产品标题}}
英文标题：{{英文标题}}
图片类型：{{图片类型}}
图片序号：{{图片序号}}
图片标签：{{图片标签}}
白底图：{{白底图}}
前端完整JSON：{{前端完整JSON}}
SKU_JSON：{{SKU_JSON}}
货源_JSON：{{货源_JSON}}
SKU_ID：{{SKU_ID}}
SKU名称：{{SKU名称}}
SKU类型：{{SKU类型}}
SKU组件_JSON：{{SKU组件_JSON}}

要求：
1. 只输出当前这一行图片任务的英文提示词。
2. 必须以白底图作为核心参考。
3. 保持商品造型、颜色、材质、比例准确。
4. 所有图片保持统一电商画风：干净、明亮、真实、适合 Temu 商品详情页。
5. 如果 SKU类型 是 combo 或 图片类型 是 组合SKU图，必须让组合内所有商品同框出现。
6. 如果是主图且商品是组合出售，主图也必须展示购买会收到的所有商品。
7. 可以有简短英文功能文字，但必须安全、中性、客观，每个标签 1-4 个词。
8. 禁止品牌名、平台名、水印、logo、医疗功效、认证、折扣、评分、二维码、夸大营销词。
9. 不要解释，不要输出 JSON，只输出一条英文提示词。
```

## 第三步：生图字段捷径

字段：`生成结果图`

类型：`Seedream 4.0`

图片输入：

```text
白底图
```

Instruction Content：

```text
{{生图提示词}}
```

建议参数：

| 参数 | 建议 |
| --- | --- |
| Aspect ratio | 1:1 |
| Declare AI-generated | Yes |
| Image Content | 白底图 |

## 回写工作台时的规则

飞书生成结果后，工作台只需要读取这些字段：

| 飞书字段 | 回写位置 |
| --- | --- |
| 链接记录ID | 找到对应 `LinkListRecord` |
| 图片类型 | 判断写入主图、轮播图还是 SKU 图 |
| 图片序号 | 判断写入第几个图片槽 |
| SKU_ID | 如果是 SKU 图，找到对应 `skuEntries[]` |
| 生成结果图URL | 生成图原 URL |
| OSS_URL | 最终采用的云端 URL，优先写入前端 |
| 状态 | 只回写 `生图完成` / `已采用` 的任务 |

前端回写建议：

- 主图：写入 `record.mainImage.editedCloudUrl`
- 轮播图：写入对应 `record.productMaterialImages[].editedCloudUrl` 或新增图片资产
- SKU 图：写入 `skuEntry.imageAsset.editedCloudUrl`
- 同时更新 `record.imageSlots`，保证预览页顺序正确

## 为什么这样更简单

- 飞书只负责一行一张图，不承担复杂数据关系。
- 前端完整 JSON 保留在一列里，后续字段不够时还能从 JSON 里解析。
- 想生成几张图就插几行，不需要预先固定 8 个图片字段。
- 想重做某一张图，只重跑对应一行。
- 同商品多行任务可以复用同一个白底图，减少重复分析和成本。
