# 数据台统一接口文档

本文档用于本地采集机器人、上传后台、未来云端网页对接「数据台」商品数据。

当前网页还未上线，本地开发地址使用：

```text
http://127.0.0.1:8000
```

上线后把示例里的 `http://127.0.0.1:8000` 替换成你的后端域名，例如：

```text
https://api.your-domain.com
```

## 接入原则

数据不要由前端或本地机器人直连数据库。

统一链路：

```text
本地云启机器人 / 上传后台 / 前端
        ↓
后端 API
        ↓
数据库
```

这样以后从本地 SQLite 迁移到云数据库时，只需要改后端数据库配置，不需要改机器人和前端逻辑。

## 数据分层

项目里目前有两层商品数据：

| 层 | 数据表 | 前端页面 | 用途 |
| --- | --- | --- | --- |
| 数据台 | `products` | 数据台 | 全量云启商品库，用于筛选、推荐、后续选品 |
| 商品池/工作池 | `product_pool_products` | 商品池 | 你手动挑出来准备处理、找货、建链接、导出的商品 |

接口规则：

- 本地机器人同步：只写入数据台，不自动进入商品池。
- 前端手动导入：写入数据台，并自动加入商品池。
- 数据台选中商品加入商品池：调用商品池接口。

## 环境变量

上线后建议配置：

```env
WORKBENCH_SYNC_TOKEN=你的机器人同步密钥
```

本地开发时如果不配置 `WORKBENCH_SYNC_TOKEN`，机器人同步接口会放行，方便测试。

前端上线时配置：

```env
VITE_API_BASE_URL=https://api.your-domain.com
```

## 统一商品字段

推荐本地机器人导出的 Excel/CSV 使用以下英文表头。云启原始中文表头也兼容，但英文表头更适合长期维护。

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `source_product_id` | 是 | string | 云启商品 ID，数据去重和更新的核心字段 |
| `title_cn` | 否 | string | 中文标题 |
| `title_en` | 否 | string | 英文标题 |
| `title` | 否 | string | 通用标题；`title_cn/title_en` 都为空时使用 |
| `main_image_url` | 否 | string | 商品主图 URL |
| `gallery_image_urls` | 否 | string/list | 轮播图 URL 列表 |
| `video_url` | 否 | string | 视频 URL |
| `source_url` | 否 | string | 云启商品详情页或来源页 URL |
| `category_path` | 否 | string/list | 类目路径，例如 `Home/Kitchen/Bowls` |
| `tags` | 否 | string/list | 标签 |
| `price_usd` | 否 | number/string | 美元价格，支持 `$12.30` |
| `gmv_usd` | 否 | number/string | GMV，支持 `1.5K`、`2万` |
| `weekly_sales` | 否 | number/string | 周销量 |
| `monthly_sales` | 否 | number/string | 月销量 |
| `review_count` | 否 | number/string | 评论数 |
| `listing_time` | 否 | string/date | 上架时间 |
| `status` | 否 | string | `active`、`deleted`、`sourced` |

兼容别名示例：

| 标准字段 | 兼容字段 |
| --- | --- |
| `source_product_id` | `product_id`、`productId`、`goods_id`、`item_id`、`spu_id`、`商品ID` |
| `title_cn` | `titleCn`、`title_zh`、`product_name_cn`、`商品标题（中文）`、`中文标题` |
| `title_en` | `titleEn`、`product_name_en`、`商品标题（英文）`、`英文标题` |
| `main_image_url` | `mainImageUrl`、`main_image`、`image_url`、`商品主图`、`主图` |
| `gallery_image_urls` | `galleryImageUrls`、`gallery_images`、`image_urls`、`carousel_images`、`商品轮播图`、`轮播图` |
| `source_url` | `sourceUrl`、`product_url`、`productUrl`、`url`、`link`、`商品链接` |
| `category_path` | `categoryPath`、`category`、`frontCategory`、`前台分类`、`类目`、`分类` |

列表字段格式：

```text
https://img.example.com/1.jpg,https://img.example.com/2.jpg
```

也支持：

```json
["https://img.example.com/1.jpg", "https://img.example.com/2.jpg"]
```

类目字段支持：

```text
Home/Kitchen/Bowls
Home > Kitchen > Bowls
Home|Kitchen|Bowls
```

## 接口总览

| 功能 | 方法 | 路径 | 用途 |
| --- | --- | --- | --- |
| 健康检查 | GET | `/api/health` | 检查后端是否运行 |
| 机器人同步云启文件 | POST | `/api/sync/yunqi/file` | 本地机器人上传云启 Excel/CSV 到数据台 |
| 前端手动导入云启文件 | POST | `/api/uploads/yunqi` | 网页导入，导入后自动加入商品池 |
| 查询商品 | GET | `/api/products` | 查询数据台或商品池 |
| 查询统计 | GET | `/api/products/stats` | 查询数据台或商品池统计 |
| 查询类目 | GET | `/api/products/categories` | 查询商品类目树 |
| 加入商品池 | POST | `/api/products/pool` | 从数据台把商品加入商品池 |
| 删除商品 | DELETE | `/api/products/{product_id}` | 软删除数据台或商品池商品 |

## 机器人同步云启文件

接口：

```text
POST /api/sync/yunqi/file
```

用途：

本地云启机器人定时采集后，把导出的 Excel/CSV 上传到云端后端。后端会：

- 保存原始上传文件。
- 解析并标准化商品。
- 按 `source_type = yunqi` + `source_product_id` 去重更新。
- 写入 `products` 数据台。
- 重建商品推荐关键词索引。
- 不自动加入商品池。

请求头：

| Header | 必填 | 说明 |
| --- | --- | --- |
| `X-Workbench-Sync-Token` | 上线后必填 | 与后端 `WORKBENCH_SYNC_TOKEN` 一致 |

也支持：

```text
Authorization: Bearer 你的同步密钥
```

Query 参数：

| 参数 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| `limit` | integer | 空 | 只导入前 N 条，测试用 |
| `rebuild_keywords` | boolean | `true` | 是否重建推荐关键词索引 |

请求体：

`multipart/form-data`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `file` | file | 云启导出的 `.xlsx` 或 `.csv` |

PowerShell 示例：

```powershell
curl.exe -X POST "http://127.0.0.1:8000/api/sync/yunqi/file" `
  -H "X-Workbench-Sync-Token: dev-token" `
  -F "file=@C:\data\yunqi_export.xlsx"
```

上线后：

```powershell
curl.exe -X POST "https://api.your-domain.com/api/sync/yunqi/file" `
  -H "X-Workbench-Sync-Token: 你的同步密钥" `
  -F "file=@C:\data\yunqi_export.xlsx"
```

成功响应：

```json
{
  "ok": true,
  "batch_id": "a1b2c3",
  "source": "yunqi-excel",
  "source_filename": "yunqi_export.xlsx",
  "excel_path": "D:/.../storage/uploads/sync_xxx_yunqi_export.xlsx",
  "file_type": "xlsx",
  "total_rows": 100,
  "imported_count": 98,
  "created_count": 20,
  "updated_count": 78,
  "failed_count": 2,
  "keyword_count": 600,
  "errors": ["row 5: missing source_product_id"]
}
```

常见错误：

| 状态码 | 含义 | 处理 |
| --- | --- | --- |
| 401 | token 错误 | 检查 `WORKBENCH_SYNC_TOKEN` 和请求头 |
| 400 | 文件可读但数据无效 | 检查表头、商品 ID、文件格式 |
| 500 | 后端处理异常 | 看后端日志 |

## 前端手动导入云启文件

接口：

```text
POST /api/uploads/yunqi
```

用途：

网页导入弹窗使用。导入后会写入：

- `products` 数据台
- `product_pool_products` 商品池

也就是说，前端手动导入代表「我要直接处理这些商品」。

请求示例：

```powershell
curl.exe -X POST "http://127.0.0.1:8000/api/uploads/yunqi" `
  -F "file=@C:\data\yunqi_export.xlsx"
```

成功响应：

```json
{
  "batch_id": "a1b2c3",
  "source_filename": "yunqi_export.xlsx",
  "file_type": "xlsx",
  "total_rows": 100,
  "imported_count": 98,
  "failed_count": 2,
  "errors": []
}
```

## 查询数据台商品

接口：

```text
GET /api/products?scope=all
```

`scope` 规则：

| scope | 查询范围 |
| --- | --- |
| `all` | 数据台 `products` |
| `pool` | 商品池 `product_pool_products` |

参数：

| 参数 | 类型 | 默认 | 说明 |
| --- | --- | --- | --- |
| `page` | integer | `1` | 页码 |
| `page_size` | integer | `10` | 每页数量，最大 100 |
| `keyword` | string | 空 | 标题、英文标题、商品 ID 模糊搜索 |
| `category` | string | 空 | 一级类目、二级类目或完整类目路径 |
| `price_min` | number | 空 | 最低价格 |
| `price_max` | number | 空 | 最高价格 |
| `sales_min` | integer | 空 | 最低销量 |
| `sales_max` | integer | 空 | 最高销量 |
| `gmv_min` | number | 空 | 最低 GMV |
| `gmv_max` | number | 空 | 最高 GMV |
| `period` | string | 空 | 最近 7 天、最近 30 天，前端使用 |
| `scope` | `all/pool` | `pool` | 查询数据台或商品池 |

示例：

```powershell
curl.exe "http://127.0.0.1:8000/api/products?scope=all&page=1&page_size=20&keyword=keychain"
```

## 删除商品

接口：

```text
DELETE /api/products/{product_id}?scope=all
DELETE /api/products/{product_id}?scope=pool
```

说明：

- `scope=all`：软删除数据台里的商品。
- `scope=pool`：只从商品池软删除，不影响数据台。

示例：

```powershell
curl.exe -X DELETE "http://127.0.0.1:8000/api/products/606112963224349?scope=pool"
```

响应：

```json
{
  "ok": true
}
```

## 本地机器人推荐流程

机器人采集完成后建议流程：

1. 导出云启数据为 `.xlsx` 或 `.csv`。
2. 表头使用本文档的统一英文字段名。
3. 上传到 `/api/sync/yunqi/file`。
4. 根据响应判断 `created_count`、`updated_count`、`failed_count`。
5. 失败时把 `errors` 写入本地日志。
6. 不要高频上传；建议按批次或定时任务上传。

Python 示例：

```python
import requests

API_BASE_URL = "http://127.0.0.1:8000"
SYNC_TOKEN = "dev-token"
file_path = r"C:\data\yunqi_export.xlsx"

with open(file_path, "rb") as file:
    response = requests.post(
        f"{API_BASE_URL}/api/sync/yunqi/file",
        headers={"X-Workbench-Sync-Token": SYNC_TOKEN},
        files={"file": ("yunqi_export.xlsx", file, "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet")},
        timeout=120,
    )

response.raise_for_status()
print(response.json())
```

## 上线前检查清单

- 后端域名已部署，并能访问 `/api/health`。
- 后端已配置 `WORKBENCH_SYNC_TOKEN`。
- 前端已配置 `VITE_API_BASE_URL` 指向后端域名。
- CORS 已允许前端域名访问后端。
- 本地机器人只调用 `/api/sync/yunqi/file`，不直连数据库。
- 上传文件里 `source_product_id` 稳定且唯一。
- 图片 URL 为可公开访问的 HTTP/HTTPS 地址。
- 定时上传有失败重试和日志记录。
