# 数据台统一 JSON 导入接口教程

本文档给以后单独编写“采集程序 / 插件 / 静态 HTML 分析程序”使用。

目标很简单：

```text
采集程序只负责拿数据并 POST JSON
后端负责识别来源、标准化、去重、入库、保留原始数据
```

## 1. 当前数据分层

项目里现在有三类商品数据视角。

| 视角 | 数据来源 | 前端位置 | 说明 |
| --- | --- | --- | --- |
| 数据台 | `products.catalog_scope = admin_catalog` | 数据台 | 管理员维护的总商品库，主要由采集程序定期更新 |
| 商品池 | `product_pool_memberships` | 商品池 | 当前用户自己的上品工作区 |
| 用户导入商品 | `products.catalog_scope = pool_only` | 只在商品池显示 | 顶部“数据导入”按钮导入的个人商品，不进入数据台 |

核心规则：

```text
数据台 = 管理员维护的总库
顶部数据导入 = 只加入当前用户商品池
/api/ingest = 给采集程序更新数据台总库
```

如果用户导入的商品已经存在于管理员数据台：

```text
不会覆盖管理员商品数据
只会把这个商品加入当前用户商品池
```

## 2. 统一接口

采集程序统一调用：

```text
POST /api/ingest
```

本地开发地址：

```text
http://127.0.0.1:8000/api/ingest
```

上线后替换成你的后端域名：

```text
https://your-api-domain.com/api/ingest
```

## 3. 认证方式

单独采集程序建议使用固定 token。

`.env` 中配置：

```env
WORKBENCH_INGEST_TOKEN=你的采集程序写入密钥
```

请求时传其中一种：

```http
X-Workbench-Ingest-Token: 你的采集程序写入密钥
```

或：

```http
Authorization: Bearer 你的采集程序写入密钥
```

网页内登录用户也可以通过 session cookie 调用，但独立程序推荐 token。

## 4. POST 前后职责

### 采集程序 POST 前负责

采集程序只做轻处理：

- 打开网页、读取 Excel、分析静态 HTML、读取插件结果
- 把每一行商品整理成 JSON 对象
- 保留原始字段
- 给出稳定来源 `source`
- 尽量提供稳定 ID 或商品 URL
- 调用 `/api/ingest`

采集程序不需要：

- 直接写 SQLite
- 判断插入还是更新
- 自己拆数据库字段
- 自己重建关键词索引
- 自己维护商品池关系
- 自己做复杂类目匹配

### 后端 POST 后负责

后端会做：

- 保存导入批次到 `ingest_batches`
- 保存每条原始数据到 `ingest_items.raw_data_json`
- 按 `source` 进入对应 adapter
- 标准化成 `products` 字段
- 按 `source_type + source_product_id` upsert
- 保存完整原始数据到 `products.raw_data_json`
- 拆分 `category_path / category_level1 / category_level2`
- 执行类目匹配或继承
- 重建关键词索引
- 返回处理结果

## 5. 请求 JSON 总格式

```json
{
  "source": "yunqi",
  "entity_type": "product",
  "mode": "upsert",
  "idempotency_key": "yunqi-us-sleeping-bag-2026-06-15",
  "context": {
    "catalog_scope": "admin_catalog"
  },
  "metadata": {
    "collector": "yunqi-rpa",
    "site": "US",
    "category": "Sleeping Bags",
    "collected_at": "2026-06-15 10:30:00"
  },
  "records": [
    {
      "商品ID": "606123456",
      "商品标题（中文）": "户外睡袋",
      "商品主图": "https://img.example.com/main.jpg",
      "商品轮播图": ["https://img.example.com/1.jpg"],
      "前台分类（中文）": "Sports & Outdoors > Camping & Hiking > Sleeping Bags",
      "月销量": "1000+"
    }
  ]
}
```

字段说明：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `source` | 是 | 数据来源，例如 `yunqi`、`1688`，后续可扩展 `dianxiaomi`、`plugin_html` |
| `entity_type` | 否 | 默认 `product`，现在所有商品采集都统一传 `product` |
| `mode` | 否 | 默认 `upsert` |
| `idempotency_key` | 建议 | 幂等键，同一批数据重复 POST 时直接返回旧批次，避免重复脏数据 |
| `context` | 否 | 本批次上下文，例如 `catalog_scope`、关联商品 ID |
| `metadata` | 否 | 采集程序信息、站点、类目、时间、文件名等 |
| `records` | 是 | 原始商品数据数组，至少 1 条 |

## 6. `catalog_scope` 规则

独立采集程序用于更新数据台，默认使用：

```json
"context": {
  "catalog_scope": "admin_catalog"
}
```

`admin_catalog` 含义：

```text
写入 products
显示在数据台
不自动进入任何人的商品池
```

`pool_only` 含义：

```text
只适合用户个人导入
不显示在数据台
通过 product_pool_memberships 显示在商品池
```

独立采集程序一般不要传 `pool_only`。

## 7. 推荐最小数据

每条记录至少要让后端能识别商品身份和标题。

### 云启商品

推荐至少有：

```json
{
  "商品ID": "606123456",
  "商品标题（中文）": "户外睡袋"
}
```

或使用标准英文字段：

```json
{
  "source_product_id": "606123456",
  "title_cn": "户外睡袋"
}
```

### 1688 商品

推荐至少有：

```json
{
  "source_url": "https://detail.1688.com/offer/123456789.html",
  "title": "睡袋工厂货源"
}
```

如果没有 `source_product_id`，后端会从 1688 URL 中解析 offer ID。

## 10. 类目处理

如果来源数据有类目：

```json
{
  "category_path": "Sports & Outdoors > Camping & Hiking > Sleeping Bags"
}
```

后端会拆成：

```text
category_path = Sports & Outdoors/Camping & Hiking/Sleeping Bags
category_level1 = Sports & Outdoors
category_level2 = Camping & Hiking
```

如果来源没有类目，但有上下文：

```json
{
  "context": {
    "related_product_id": "已有 products.id"
  }
}
```

后端会尝试继承关联商品类目。

如果既没有来源类目，也没有上下文：

```text
不阻止导入
ingest_items.category_match_status = missing
后续可再补类目
```

## 11. Upsert 和幂等

商品去重更新规则：

```text
source_type + source_product_id
```

例如：

```text
yunqi + 606123456
1688 + 123456789
```

再次 POST 同一个商品：

```text
更新原记录
不重复插入
created_at 保留
updated_at 更新
```

批次幂等规则：

```json
"idempotency_key": "yunqi-us-sleeping-bag-2026-06-15"
```

同一个 `source + entity_type + idempotency_key` 再次提交时，后端会返回旧批次：

```json
{
  "idempotent_replay": true
}
```

建议采集程序每次生成稳定批次键，例如：

```text
{source}-{site}-{category}-{date}-{page_or_export_time}
```

```python
response.raise_for_status()

```

## 15. 从 Excel/CSV 转成 records

采集程序可以自己读取表格，然后原样 POST。

```python
import pandas as pd

def table_to_records(path: str) -> list[dict]:
    if path.lower().endswith(".xlsx"):
        df = pd.read_excel(path)
    else:
        df = pd.read_csv(path, encoding="utf-8-sig")

    records = []
    for index, row in df.iterrows():
        record = {}
        for column in df.columns:
            value = row[column]
            if pd.isna(value):
                continue
            record[str(column).strip()] = value
        record["source_row_index"] = index + 1
        records.append(record)
    return records
```

然后：

```python
payload = {
    "source": "yunqi",
    "idempotency_key": "yunqi-export-2026-06-15-1030",
    "context": {"catalog_scope": "admin_catalog"},
    "metadata": {"original_filename": "yunqi_export.xlsx"},
    "records": table_to_records(r"C:\data\yunqi_export.xlsx"),
}
```

## 17. 后续新增来源怎么做

采集程序仍然只 POST：

```json
{
  "source": "dianxiaomi",
  "records": []
}
```

后端再新增一个 adapter：

```text
source = dianxiaomi
-> dianxiaomi product adapter
-> 标准化 products
-> upsert
```

不要为每个来源新建一个接口。统一入口始终是：

```text
POST /api/ingest
```

## 18. 常见错误

| 错误 | 原因 | 处理 |
| --- | --- | --- |
| 401 | token 不对或未登录 | 检查 `WORKBENCH_INGEST_TOKEN` 和请求头 |
| 400 `暂不支持的数据入口` | `source` 还没有后端 adapter | 新增对应来源 adapter |
| 400 `missing source_product_id` | 云启记录缺少商品 ID | 检查表格字段或字段别名 |
| 400 `missing product title` | 记录没有标题 | 至少传 `title`、`title_cn` 或来源标题字段 |
| `partial` | 有些行失败 | 查看响应 `items[].errorMessage` |
| 数据台看不到 | 可能传了 `catalog_scope=pool_only` | 数据台只显示 `admin_catalog` |

## 19. 验证数据是否进数据台

调用：

```text
GET /api/products?scope=all&page=1&page_size=20
```

`scope=all` 当前含义是：

```text
只看管理员总库 admin_catalog
```

如果要看当前用户商品池：

```text
GET /api/products?scope=pool&page=1&page_size=20
```

## 20. 独立采集程序推荐流程

```text
1. 采集来源页面 / 表格 / HTML
2. 转成 records 数组
3. 生成 idempotency_key
4. POST /api/ingest
5. 检查 batch.status
6. 记录 failed items
7. 定时任务下次继续跑
```

推荐默认配置：

```json
{
  "entity_type": "product",
  "mode": "upsert",
  "context": {
    "catalog_scope": "admin_catalog"
  }
}
```

一句话总结：

```text
你的独立程序只需要 POST 原始商品 JSON。
字段标准化、去重、更新、类目、数据台展示都交给后端。
```
