# 小规模上线测试部署清单

目标：先支持少量真实账号跑通「登录隔离 -> 数据台/商品池 -> 链接列表 -> 主图异步生成 -> Excel 导出」闭环，不追求一开始就高并发。

## 推荐组件

- 后端：FastAPI + Uvicorn
- 前端：Vite build 后由 Nginx 托管
- 数据库：小规模测试可继续 SQLite；多人稳定测试建议迁移 PostgreSQL
- 队列：Redis
- Worker：`backend/scripts/run_visual_queue_worker.py`
- 图片云存储：阿里云 OSS
- API 渠道：FluAPI / OpenAI-compatible

## 必须启动的进程

1. Redis
2. 后端 API
3. 视觉生图 worker
4. 前端静态站点

本地验证命令：

```bash
cd D:\learning\temu_listing_workbench
python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000
python backend/scripts/run_visual_queue_worker.py
cd frontend
npm run build
```

## 小规模测试建议参数

```env
VISUAL_QUEUE_REDIS_ENABLED=1
VISUAL_USER_CONCURRENCY_LIMIT=1
VISUAL_TEAM_CONCURRENCY_LIMIT=3
VISUAL_QUEUE_DRAIN_MAX_JOBS=2
VISUAL_QUEUE_MAX_RETRIES=2
VISUAL_QUEUE_RETRY_DELAY_SECONDS=30
```

解释：

- 单用户同时只跑 1 个生图任务，避免误点导致 API 费用爆炸。
- 一个管理员团队最多 3 个 active 任务，适合 2-5 人试用。
- Worker 每轮最多处理 2 个任务，降低服务器瞬时压力。
- 失败自动重试 2 次，仍失败进入死信队列，方便排查。

## 上线前冒烟测试

1. 登录普通用户，确认只能看到自己的商品池和链接列表。
2. 登录管理员，确认能看到人员管理、API 配置和队列配置。
3. 从数据台加入 1 个商品到商品池。
4. 选择 SKU，录入链接列表。
5. 在链接列表创建主图全量生成任务。
6. 打开任务队列，确认出现 queued/running/completed 状态。
7. 生成完成后确认商品预览图更新。
8. 导出 Excel，确认标题、图片 URL、SKU 顺序和产品属性列存在。

## 暂不适合上线前忽略的事项

- API Key 不要写死在前端。
- `.env` 不要提交到 Git。
- OSS Bucket 必须确认图片能被店小秘以公网 HTTP/HTTPS 访问。
- Redis 和后端进程要配置守护进程或进程管理器，否则服务器重启后任务不会继续消费。
- SQLite 适合小规模测试，不适合大量用户同时写入；进入商用前建议迁移 PostgreSQL。
