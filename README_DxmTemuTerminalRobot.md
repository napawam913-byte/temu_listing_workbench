# 店小秘 Temu 终端机器人

这是纯 Python 终端版机器人，最终打包为命令行 exe 使用。

## 技术选型

- Python Playwright CDP：连接固定 Chrome 的 `http://127.0.0.1:9222`。
- 固定登录目录：`work/chrome-profile`，保留店小秘登录态。
- 模块化 RPA：当前支持 `基本信息` 商品属性补全、`产品信息` 标题翻译/图片下载、`变种信息` 固定规则填写、`产品描述` 批量传图、`运输信息` 固定规则填写。
- 店小秘 Temu 基本信息适配器：固定代码识别基本信息模块内的红星必填、Ant 下拉框、复选框组、标题输入框和商品图片。
- 人为点击模拟：读取候选项前会滚动到基本信息模块内的控件、鼠标移动点击、等待下拉/复选浮层真实显现。
- 图片下载：只收集 `产品信息` 模块范围内的商品图片链接并保存到指定文件夹。
- Python 翻译器：当前 `产品信息` 模块标题修正不使用老张 API，不使用 AI，只做普通翻译。
- 图片下载：只收集 `产品信息` 模块范围内的图片，保存到你指定的文件夹，并生成 `manifest.json`。
- 变种信息固定填写：仓库选 `哥伦比亚飞特`，库存填 `0`，申报价格填 `300`，尺寸长宽高填 `10/10/10`，重量填 `200`，并点击对应批量/同步图标。
- 产品描述批量传图：进入产品描述编辑器后，点击 `批量操作 > 批量传图 > 选择图片 > 本地上传`，上传本地图片并点击保存。
- 运输信息固定填写：承诺发货时效选择 `16个工作日内发货`，运费模板选择 `哥伦比亚`。
- 红线规则：基本信息自动填写前会检测 `液体`、`带电`、`棉`；命中后停止自动填写并要求人工复核。
- 终端实时日志：每一步操作都会打印动作日志。

## 运行

```powershell
.\dist\DxmTemuTerminalRobot\DxmTemuTerminalRobot.exe
```

当前交付为终端 exe。源码和打包脚本保留用于后续维护；打包产生的 `build`、`dist`、`.spec`、`__pycache__` 已清理。

## 使用顺序

1. 双击或在终端运行 `dist\DxmTemuTerminalRobot\DxmTemuTerminalRobot.exe`。
2. 选择 `1. 一键准备环境`。
3. 在固定 Chrome 登录店小秘。
4. 打开 Temu 半托管商品编辑页，URL 需要包含 `edit?id=...`。
5. 选择 `1. 一键准备环境`，它会合并执行：启动固定 Chrome、连接 CDP、打开/绑定商品编辑页、查看标签页、扫描模块边界。
6. 选择 `2. 扫描基本信息商品属性`，确认识别到 `颜色、类型、主体材质、适用年龄段` 等字段。
7. 选择 `3. AI补全基本信息红星必填商品属性`，机器人只补全 `基本信息 > 产品属性` 里的红星必填空字段。
8. 选择 `4. 读取产品信息中英文标题`，确认能读取 `产品标题` 和 `英文标题`。
9. 选择 `5. Python 翻译产品信息中英文标题`，机器人会判断：`产品标题` 必须中文，`英文标题` 必须英文，不符合时用 Python 翻译器翻译并真实点击输入框填写。
10. 选择 `6. 下载产品信息模块图片到指定文件夹`，输入保存路径；留空则保存到 `work/downloads/产品信息/时间戳`。
11. 选择 `7. 填写变种信息：仓库/库存/申报价格/尺寸/重量`，执行固定规则并打印每一步动作日志。
12. 选择 `8. 产品描述：批量传图并保存`，输入图片文件夹；留空默认使用程序目录下的 `桌面` 文件夹。
13. 选择 `9. 填写运输信息：16个工作日/哥伦比亚运费模板`，执行固定规则并打印每一步动作日志。

## 规则

- `基本信息` 模块已恢复商品属性补全：扫描和补全只作用于 `基本信息 > 产品属性` 区域，不会跳到其他模块。
- 商品属性补全只处理红星必填且当前为空/请选择的字段；已填好的 `颜色、类型、适用年龄段` 不会重复改。
- 商品属性补全使用老张 API / `gpt-4o-mini` 决策候选项，但点击、下拉读取、选项匹配由固定 Python RPA 执行。
- 商品属性补全会动态循环：每填一个字段后重新扫描，处理因页面变化新出现的红星必填字段，最多 8 轮。
- 材质正向规则：候选项里有 `聚酯纤维` / `涤纶（聚酯纤维）` / `涤纶` 时优先选择；如出现成分/含量字段且材质为聚酯纤维或涤纶，填写 `100`。
- 类型固定规则：`基本信息 > 产品属性 > 类型` 固定选择 `详见商品详情`，这条规则不交给 AI 决策；若当前商品页面未渲染该字段则记录跳过。
- 适用年龄段规则：根据 `产品标题`、`英文标题`、`产品分类`、`类型` 判断，再从页面实际下拉候选中选择；儿童/玩具/派对礼品语义优先 `3+`，婴幼儿语义优先低龄候选，成人/办公装饰语义优先更高年龄或 `不适用`。
- 商品属性选择尽量模糊边界，商品信息不明确时不选 `ABS`、`EVA`、`PC`、`PET`、`PU`、`PVC`、`TPU` 等过于专业的细分材质。
- `变种信息` 固定填写不使用 AI，只按固定 Python RPA 规则执行：`哥伦比亚飞特`、库存 `0`、申报价格 `300`、尺寸 `10/10/10`、重量 `200`。
- `产品描述` 批量传图不使用 AI；文件选择通过 Playwright 设置本地文件，避免依赖 Windows 文件选择框坐标。
- `运输信息` 固定填写不使用 AI，只限定在 `#shipmentInfo` 模块内执行：点击 `16个工作日内发货`，打开运费模板下拉并点击 `哥伦比亚`，最后触发失焦让页面校验恢复。
- `产品标题`、`英文标题` 属于 `产品信息` 模块，不属于页面最上方的 `基本信息` 模块。
- 红线规则为 `液体`、`带电`、`棉`。标题、英文标题、分类、当前已选值命中红线时，基本信息自动填写停止。
- 候选项命中红线时会被过滤，不会交给 AI 选择，也不会点击填写。
- `不带电`、`不带电池`、`不含电池`、`无需供电` 等否定表达不会被误判为 `带电`。
- 标题修正会先读取当前页面 `产品标题` 和 `英文标题`，判断语言后再用 Python 翻译器翻译；如果已经符合要求就不会改。
- 菜单 `12` 可测试 Python 翻译器连通性。
- 菜单 `1` 是准备环境合并步骤，执行原来的 `启动 Chrome`、`连接 CDP`、`打开/绑定商品编辑页`、`查看标签页`、`扫描模块边界`。
- 页面填写由固定 Python RPA 完成：滚动到 `产品信息` 模块，真实点击输入框，Ctrl+A，写入，触发 input/change/blur。
- 图片下载会滚动 `产品信息` 模块触发懒加载，收集 `<img>`、懒加载属性、CSS 背景图、blob/data 图片。
- 图片下载只保存宽高都不小于 `180x180` 的图片，并过滤店小秘页面 assets 里的 icon/sprite/hover 控件背景；类似 `85x16`、`28x28`、`20x20` 的小图不会再下载。
- 每次图片下载会在目标文件夹生成 `manifest.json`，记录来源链接、保存文件、失败原因。

## 不要删除

- `work/chrome-profile`：固定 Chrome 登录态。
- `work/state/laozhang-api.json`：老张 API 配置。
- `work/extracted_terminal_app.pyc`：旧完整程序模块，源码维护和重新打包需要它；最终 exe 内部也已嵌入一份。
